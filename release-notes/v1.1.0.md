# Release Notes — v1.1.0 (2026-06-15)

## Highlights
- **Codex generation workspace**: editors can open `/generate-story` from `/manage`, submit a story idea, optional target date, and up to three reference images, then watch live Sprite/Codex job logs.
- **Managed story-agent jobs**: jobs are stored in D1, scoped to the requesting editor, stream events over SSE, accept feedback messages while running, support cancellation, and link completed jobs back to the generated story.
- **Automation API**: `X-Story-Token` protected endpoints now support calendar lookup, media upload, JSON story creation, and partial JSON story updates for the bundled generation skill and Sprite runner.
- **Generation skill and media pipeline**: the repo includes the local `generate-story` Codex skill and scripts for date selection, image/video generation, upload, and publish/update workflows. Defaults use Replicate with `google/nano-banana-2` images and WAN image-to-video, with OpenAI media modes still available.
- **Security and reliability hardening**: authentication and runner callbacks use timing-safe token checks, hashed callback tokens, stricter upload/media validation, no-store headers for sensitive pages, authenticated cache refresh, and a browser-like Sprite runner user agent for Cloudflare Browser Integrity Check compatibility.

## Changes
- Added `public/generate-story.html` and routed `/generate-story` / `/generate-story.html` as editor-only pages.
- Added story-agent APIs under `/agent/jobs/*` for browser clients and `/api/agent/jobs/*` for runner callbacks.
- Added `db/story_agent_tables.sql` for story-agent jobs, reference images, events, and feedback messages.
- Added `src/agent.ts`, `src/storyAgentRunner.ts`, and `src/security.ts` to manage Sprite launches, runner bootstrap, callback authorization, events, cancellation, and shared token helpers.
- Added `POST /api/media`, `POST /api/stories`, `PUT /api/stories/:id`, and `GET /api/stories/calendar` for token-authenticated automation.
- Updated story create/update/delete flows to handle video objects alongside image objects and to clean up replaced media.
- Added media upload size/type checks, media key validation, HTML security headers, and stricter no-store/no-referrer behavior on authentication-sensitive routes.
- Added the local `.codex/skills/generate-story` skill and helper scripts for generating and publishing stories with image and video media.
- Updated README/AGENTS documentation for automation workflows, story-agent setup, and release operation notes.
- Updated Wrangler to `^4.58.0` and refreshed `package-lock.json`.
- Set `PUBLIC_VIEW` to `true` in `wrangler.jsonc`.

## Upgrade Notes
- Node.js 20+ is now required for the locked Wrangler 4.58.0 toolchain.
- Run the new story-agent schema before enabling Codex generation:

```bash
wrangler d1 execute bedtime-stories --file db/story_agent_tables.sql
```

- Configure these Worker secrets before using `/generate-story`:
  - `SPRITES_API_TOKEN`
  - `STORY_AGENT_ALLOWED_EMAILS`
  - `STORY_API_TOKEN`
- Keep the configured Sprite updated with this repo and the bundled `generate-story` skill. The default Sprite name is `bedtime-stories`; override with `STORY_AGENT_SPRITE_NAME` if needed.
- If Cloudflare Browser Integrity Check blocks Sprite callbacks with Error 1010, deploy this Worker version and selectively bypass Browser Integrity Check for the authenticated automation API paths only if the browser-like user agent is still blocked.
- Leave `STORY_AGENT_ALLOWED_EMAILS` unset to keep the high-cost story-generation surface disabled.
- Existing story rows do not need migration for this release, assuming the v1.0.3+ `video_url` column is already present.
